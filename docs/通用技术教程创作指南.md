# 通用技术教程创作指南

## 🎯 核心理念

### 1. 以「为什么」为中心
- 不是 "用 X 实现 Y"，而是 "为什么 X 能解决 Z 问题？"
- 说明技术出现的背景和驱动力
- 对比传统方案的痛点
- 解释设计决策的权衡

### 2. 认知阶梯
从浅入深的五层结构：
- **直观理解**：比喻、类比、可视化
- **概念模型**：架构图、核心抽象
- **实现机制**：关键算法、数据流
- **生产实践**：最佳实践、常见陷阱
- **扩展思考**：未来演进、相关技术

### 3. 对话式教学
- 预设读者疑问："你可能会想..."
- 揭示设计权衡:"这里面临的选择是..."
- 引导主动思考:"如果是你会如何设计?"

## 📐 教程结构模板

### 标准五部分结构

**1. 导览**
```markdown
## 📋 本章导览
- 你将掌握的技能
- 预计学习时间
- 核心挑战
```

**2. 理解原理**
```markdown
## 📚 理解 [技术名称]

### 为什么需要这个技术？
- 现实场景描述
- 传统方案的问题
- 新方案如何解决

### 核心概念
- 关键抽象和数据结构
- 工作机制（配图说明）
- 实验：观察真实示例
```

**3. 设计方案**
```markdown
## 🔧 设计我们的实现

### 架构思考
- 如何表示核心概念？
- 如何处理错误？
- 模块如何组织？

### 架构图
[展示组件关系和数据流]

### 设计权衡
- 方案对比
- 选择理由
```

**4. 代码实现**
```markdown
## 💻 代码实现

### 添加依赖
### 实现核心模块
### 实现命令接口

[代码中包含详细注释，解释"为什么"而非"是什么"]
```

**5. 验证总结**
```markdown
## ✅ 测试和验证
- 运行单元测试
- 手动测试
- 与真实系统对比

## 🤔 深入思考
- 常见问题解答
- 设计模式分析

## 📝 总结与展望
- 学到了什么
- 下一步方向
```

## 🎨 表达技巧

### 1. 可视化
- **架构图**：清晰标注组件职责和数据流向
- **对比表格**：方案优劣和适用场景
- **ASCII 图**：目录结构、流程图

### 2. 代码注释原则
```rust
// ❌ 不好：只说"是什么"
let hash = compute_hash(data);

// ✅ 好：解释"为什么"
// Git 使用 SHA-1 确保内容完整性
// 相同内容产生相同哈希，实现去重
let hash = compute_hash(data);
```

### 3. 互动元素
```markdown
<details>
<summary>💡 思考：为什么要这样设计？</summary>
[引导性分析，而非标准答案]
</details>
```

### 4. 层次化问答
- **初学者**："为什么不用更简单的方法？"
- **进阶者**："如何权衡性能和复杂度？"
- **专家**："如何应对未来的演进？"

## 🌟 优秀教程的标志

**做到了什么：**
1. **问题共鸣** - 读者感到"这正是我的困惑"
2. **顿悟时刻** - 复杂概念变得清晰自然
3. **实践信心** - 读者相信自己能独立解决问题
4. **思维提升** - 不仅学会技术，更学会思考方法

**避免的陷阱：**
- ❌ 特性罗列（缺乏为什么）
- ❌ 过度简化（丢失关键细节）
- ❌ 假设背景（使用未解释的术语）
- ❌ 缺乏场景（纯抽象描述）

---

**记住核心原则**：优秀的教程不是信息传递，而是**认知重构**。让读者理解何时使用、为什么有效、如何应对变化。

## 📁 推荐文档结构

基于实践经验，推荐以下文档组织方式：

```
docs/
├── 01-concepts.md           # 核心概念和原理
├── 02-project-plan.md       # 项目规划和架构设计
├── 03-tech-stack.md         # 技术选型说明
└── tutorials/               # 分步教程
    ├── README.md           # 教程索引和学习路径
    ├── tutorial-01-xxx.md  # 教程1：基础功能
    ├── tutorial-02-xxx.md  # 教程2：核心系统
    ├── tutorial-03-xxx.md  # 教程3：高级特性
    └── ...
```

### 各文档的作用

**01-concepts.md** - 原理概念
- 技术的本质和工作机制
- 核心数据结构和算法
- 关键术语解释
- 适合独立阅读，建立全局认知

**02-project-plan.md** - 项目规划
- 整体架构设计
- 模块划分和职责
- 开发路线图和里程碑
- API 设计草案
- 适合开发前规划和全局把控

**03-tech-stack.md** - 技术选型
- 为什么选择这些技术？
- 各技术的优劣对比
- 依赖库的选择理由
- 替代方案分析
- 适合理解技术决策

**tutorials/** - 分步教程
- 每个教程一个独立功能
- 循序渐进，相互衔接
- 包含完整的实现和测试
- 适合边学边做
